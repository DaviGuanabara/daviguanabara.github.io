# Defense Resume Design Specification

## Purpose

This document defines the product design for the first production-quality
Defense Resume generated by the Resume Engineering System (RES).

It is not the final resume.

It is the design specification for the resume that should eventually target
roles such as:

- Senior AI / Machine Learning Engineer
- Applied AI Engineer
- Autonomous Systems Engineer
- Machine Learning Engineer for Defense / Aerospace

Representative employers include:

- Lockheed Martin Artificial Intelligence Center (LAIC)
- RTX
- Northrop Grumman
- Anduril
- Palantir
- Boeing Defense

This document answers one question:

> If this were the strongest truthful AI/ML defense resume that could be
> generated from the candidate's background, what should it look like?

## Design Principles

### 1. Truthfulness over optimization

The resume should be strong, but never stronger than the evidence available.

Do not invent:

- professional employers
- cloud platforms
- production deployments
- leadership titles
- certifications
- security clearances
- metrics that do not exist
- tools not grounded in the repository facts

### 2. Engineering-first positioning

The candidate should read as an engineer who builds AI systems, not only as a
researcher who studies them.

Preferred signals:

- modular software design
- simulation infrastructure
- reinforcement learning systems
- multi-agent reasoning
- autonomous systems
- aerospace / operational context
- experimentation discipline
- technical communication

### 3. Defense alignment

The resume should sound compatible with defense and aerospace organizations.

That means emphasizing:

- autonomy
- decision-making under uncertainty
- simulation
- sensing and coordination
- system architecture
- reliability of experimentation
- software built around operational scenarios

### 4. ATS discipline

The document should remain ATS-friendly and reviewer-friendly:

- one-column main body
- explicit section headings
- plain labels
- high information density
- consistent bullets
- no decorative visual tricks

## Target Resume Shape

Preferred one-page version:

```text
Header
Professional Summary
Technical Skills
Selected AI Projects
Education
Selected Publication
```

Fallback two-page version:

```text
Page 1:
Header
Professional Summary
Technical Skills
Selected AI Projects

Page 2:
Education
Selected Publication
Links
```

### Product Decision

The first target should still aim for one page.

Reason:

- defense and AI engineering resumes benefit from fast scanning
- current factual scope is still compact
- forcing one page improves prioritization

Risk:

- a one-page limit may compress nuance
- publication and education depth may become secondary

Mitigation:

- keep only one selected publication
- use high-density but readable bullets
- move links to the header when possible

## Section Design

## Header

### Why it exists

The header establishes professional identity, location, and contact path in
less than five seconds.

### Objective

Signal "AI / ML engineer for autonomous and aerospace systems" before the
reviewer reaches the first project bullet.

### Maximum length

2 visual lines, excluding whitespace.

### Writing style

Minimal, direct, professional.

### Information hierarchy

1. Name
2. Target role label
3. Email, phone, location
4. GitHub, LinkedIn, Lattes if space allows

### Expected engineering tone

Clean and unembellished.

### Good example

`Davi Guanabara Aragao`

`AI / Machine Learning Engineer | Autonomous Systems | Reinforcement Learning`

### Bad example

`Passionate AI innovator building the future of defense technology`

Why bad:

- sounds promotional
- not evidence-based
- weakens credibility

## Professional Summary

### Why it exists

This section translates a fragmented background into a single coherent
 professional identity.

### Objective

Position the candidate as:

- an AI engineer
- a Ph.D. researcher at ITA
- focused on reinforcement learning and intelligent systems
- building simulation-centered software for autonomous aerospace applications

### Maximum length

3 to 4 lines.

Approximately 55 to 85 words.

### Writing style

Natural US English.

No clichés, no motivational phrasing, no inflated claims.

### Information hierarchy

1. Current identity and technical focus
2. Systems domain
3. Engineering strengths
4. Application context

### Expected engineering tone

Calm, precise, technically literate.

### Good example

`AI engineer and Ph.D. researcher at the Aeronautics Institute of Technology (ITA) building intelligent systems for autonomous and aerospace applications. Focused on reinforcement learning, multi-agent systems, simulation, and decision-making under uncertainty, with hands-on work in modular Python architectures, experimentation pipelines, and applied machine learning for UAV and digital-twin-style operational scenarios.`

### Bad example

`Results-driven AI professional with strong passion for innovation, teamwork, and solving complex challenges in cutting-edge environments.`

Why bad:

- generic HR language
- says almost nothing specific
- could describe thousands of candidates

## Technical Skills

### Why it exists

Defense recruiters often scan for technical fit before they read projects in
detail.

### Objective

Translate the factual skill base into categories that sound like engineering
capabilities rather than a loose keyword dump.

### Maximum length

4 to 6 compact groups.

Each group should fit on one line when possible.

### Writing style

Categorical, explicit, restrained.

### Information hierarchy

1. Programming
2. Machine Learning
3. Autonomous Systems
4. Simulation
5. Software Engineering
6. Scientific Computing

### Expected engineering tone

Taxonomic and credible.

### Good example

`Programming: Python, PyTorch`

`Machine Learning: Reinforcement learning, supervised learning, deep learning, attention mechanisms`

### Bad example

`Tools: AI, ML, data, robotics, software, research`

Why bad:

- too vague
- not factual enough
- not actionable for screening

### Ideal skill grouping for this resume

This is the desired product shape, pending factual validation:

- Programming: Python, PyTorch
- Machine Learning: Reinforcement learning, supervised learning, deep learning, attention mechanisms
- Autonomous Systems: Multi-agent systems, autonomous UAV decision-making, behavior trees
- Simulation: Modeling and experimentation in simulated environments, operational simulation, decision-making under uncertainty
- Software Engineering: Modular architecture, experimentation pipelines, evaluation workflows
- Scientific Computing: Geo-temporal modeling

### Missing information

The current factual base does not explicitly confirm some items above as
separate structured skills. They are inferred from project wording.

That means the final implementation should only include categories whose terms
are supported by project or experience evidence.

## Selected AI Projects

### Why it exists

Projects are the core proof section of this resume.

### Objective

Show how research-oriented work becomes engineering work:

- systems built
- architectures designed
- simulations created
- learning pipelines implemented
- autonomy problems addressed

### Maximum length

4 to 5 projects.

2 to 3 bullets per project.

Each bullet should be one tight sentence.

### Writing style

Engineering-first, not publication-first.

Each bullet should answer:

1. What was built?
2. For what problem?
3. Using which technical approach?
4. Why is it interesting in a defense / autonomy context?

### Information hierarchy

1. Most defense-aligned software engineering project
2. Strongest applied AI autonomy project
3. Strongest perception / sensing project
4. Strong systems / simulation project
5. Broader ML framework project

### Expected engineering tone

Concrete, systems-oriented, technically confident.

### Good bullet pattern

`Designed [system] in [technology/context] to solve [technical problem], enabling [engineering value or operational relevance].`

### Bad bullet pattern

`Studied reinforcement learning for UAVs.`

Why bad:

- does not show engineering
- no artifact
- no technical structure
- no operational relevance

## Project Narrative Order

Ideal order for the defense resume:

1. SkyWeaver
2. Deep Reinforcement Learning for Cooperative Loyal Wingmen
3. ThreatSense
4. Urban Airspace Management
5. Wildfire AI Framework

### Product rationale

This order moves from:

- software platform
- autonomy/control
- perception/fusion
- airspace/simulation
- general ML infrastructure

That sequence feels coherent for defense AI reviewers because it broadens from
core system architecture into applied mission and experimentation capability.

### Critical factual note

SkyWeaver is not currently represented as a validated project in repository
data. It appears only as an architectural example in the design guidance.

Therefore:

- SkyWeaver should be treated as desired top-of-page content
- but it must not enter the actual resume until its factual details are
  confirmed

If SkyWeaver cannot be validated, then the strongest truthful order becomes:

1. Deep Reinforcement Learning Applied to Loyal Wingmen Drones
2. ThreatSense
3. Airspace Simulation and Management for UAVs
4. Wildfire AI Framework

## Project-by-Project Design

## 1. SkyWeaver

### Intended role in the resume

This should be the flagship engineering project if it is real and factually
supportable.

### Why it would exist

It would anchor the resume in software architecture, not only ML.

### Desired positioning

Present primarily as a software engineering platform for simulation and
operational reasoning.

### Emphasis

- Python
- event-driven runtime
- modular architecture
- reactive execution
- dependency graph
- GIS workflows
- strongly typed design
- digital twin style operational simulation

### Ideal tone

Architecture and systems design.

### Good bullet examples

`Designed a modular event-driven runtime in Python for reactive execution of GIS-driven simulation workflows supporting autonomous-system scenarios.`

`Built a dependency-graph-based execution model to propagate state changes across simulation components, improving composability for digital-twin-style operational experiments.`

`Structured the platform around strongly typed interfaces and modular services to support maintainable experimentation in geospatial and autonomy-focused workflows.`

### Bad bullet examples

`Worked on a GIS project for simulation.`

`Created a useful platform for many scenarios.`

### Missing information

Currently missing from repository facts:

- confirmed project existence
- exact title
- date range
- verified stack beyond Python/GIS-style wording
- whether digital twin is factual or only directional wording

### Design conclusion

Do not include until verified.

## 2. Deep Reinforcement Learning for Cooperative Loyal Wingmen

### Intended role in the resume

Primary applied AI and autonomy project.

### Why it exists

This is the strongest current proof that the candidate builds ML systems for
autonomous aerospace scenarios.

### Desired positioning

Applied AI for cooperative autonomous UAV decision-making.

### Emphasis

- reinforcement learning
- PyTorch
- cooperative agents
- autonomous UAVs
- simulation
- decision-making under uncertainty
- neural architectures for policy learning

### Good bullet examples

`Developed a deep reinforcement learning system for threat-engagement decision-making in simulated Loyal Wingmen air-combat scenarios.`

`Built a UAV simulation environment for autonomous decision-making under uncertainty, enabling policy-learning experiments for cooperative agent behavior.`

`Implemented neural policy-learning workflows in PyTorch for dynamic control problems involving autonomous aerial agents.`

### Bad bullet examples

`Researched reinforcement learning for drones.`

`Used AI in UAV simulations.`

### Repeated concepts to avoid

- repeating "simulation" in every bullet
- repeating "autonomous UAVs" in every bullet
- mixing this project with ThreatSense perception language

## 3. ThreatSense

### Intended role in the resume

Perception, sensing, and multi-agent information-sharing project.

### Why it exists

This project complements Loyal Wingmen by shifting from control to perception.

### Desired positioning

Multi-agent perception and sensor-fusion-oriented AI for simulated autonomous
systems.

### Emphasis

- shared observations
- attention mechanisms
- sensor fusion
- multi-agent perception
- simulation

### Good bullet examples

`Designed a perception-oriented AI workflow to extend situational awareness for simulated Loyal Wingmen agents through shared multi-agent observations.`

`Applied attention-based neural architectures to model information sharing and observation fusion across cooperating autonomous agents.`

`Integrated perception-focused learning components into a simulation environment to study multi-agent sensing and coordination behavior.`

### Bad bullet examples

`Improved awareness in drones.`

`Used attention for sensing.`

### Repeated concepts to avoid

- restating the Loyal Wingmen RL framing
- overusing the phrase "situational awareness"

## 4. Urban Airspace Management

### Intended role in the resume

Systems and simulation breadth project with clear aerospace relevance.

### Why it exists

It broadens the resume beyond combat autonomy into airspace coordination and
operational modeling.

### Desired positioning

Simulation and distributed decision-making for low-altitude urban UAV
operations.

### Emphasis

- multi-UAV coordination
- urban airspace
- simulation
- distributed decision-making
- traffic management
- operational safety
- AAM support

### Good bullet examples

`Developed simulation models for coordinating multiple UAVs in low-altitude urban airspace scenarios.`

`Explored distributed decision-making approaches for airspace management, operational safety, and traffic coordination in autonomous aerial operations.`

`Applied multi-agent and computational modeling concepts to support Advanced Air Mobility simulation studies.`

### Bad bullet examples

`Worked on urban air mobility concepts.`

`Studied airspace for UAVs.`

## 5. Wildfire AI Framework

### Intended role in the resume

Show reusable ML engineering capability outside a single mission scenario.

### Why it exists

It proves framework thinking, experimentation discipline, and modular
engineering.

### Desired positioning

Machine learning framework and experimentation infrastructure.

### Emphasis

- modular ML framework
- experimentation
- evaluation pipelines
- logging
- benchmarking
- reproducibility
- geo-temporal data

### Good bullet examples

`Built a modular machine learning framework for supervised learning on geo-temporal data, supporting repeatable experimentation workflows.`

`Implemented evaluation, logging, and benchmarking pipelines to compare model behavior across controlled training runs.`

`Structured the framework around modular learning components to improve maintainability and reproducibility in applied ML experiments.`

### Bad bullet examples

`Created a framework for machine learning.`

`Used attention mechanisms on data.`

### Repeated concepts to avoid

- overemphasizing research novelty
- repeating generic “modular” language with no artifact attached

## Education

### Why it exists

The educational trajectory is unusually relevant to defense and research roles.

### Objective

Signal formal progression into intelligent systems, aeronautics, data science,
and doctoral research.

### Maximum length

3 to 5 entries.

Ideally only one detail line for the most relevant degrees.

### Writing style

Compact, factual, ordered by relevance or reverse chronology.

### Information hierarchy

1. Ph.D. at ITA
2. Data Science specialization at ITA
3. Professional M.S. at ITA
4. Prior education only if space remains

### Expected engineering tone

Credentialed and focused, not academic for its own sake.

### Good example

`Ph.D. in Electronic Engineering and Computing, Aeronautics Institute of Technology (ITA), 2025-Present`

### Bad example

`Extensive academic journey across many excellent institutions`

Why bad:

- interpretive rather than factual

## Publications

### Why it exists

This section is valuable only if there is at least one real publication that
reinforces the engineering narrative.

### Objective

Show that the candidate can also communicate technical work rigorously.

### Maximum length

1 selected publication.

2 at most if the resume becomes two pages.

### Writing style

Clean bibliographic style with strong title readability.

### Information hierarchy

1. Most relevant AI / autonomy / simulation publication
2. Venue or status
3. Year

### Expected engineering tone

Supportive evidence, not a research dump.

### Good example

`Author, "Title of publication," Venue/Status, Year.`

### Bad example

Dumping a long list of citations with no prioritization.

### Missing information

No structured publication data currently exists in the RES data model used by
the defense profile.

Therefore this section should not be implemented until one real publication is
selected and validated.

## Repeated Concepts To Reduce

The current repository wording repeats a few concepts too often:

- simulation
- reinforcement learning
- multi-agent systems
- autonomous UAVs
- decision-making under uncertainty

These are important, but repetition reduces perceived sophistication.

The final resume should distribute them strategically:

- Loyal Wingmen: control and decision-making
- ThreatSense: perception and shared observations
- Urban Airspace: coordination and operational modeling
- Wildfire: framework and experimentation
- SkyWeaver: architecture and execution model

## Information Missing From the Current Factual Base

The following items are needed to reach the ideal product:

### High priority missing facts

- confirmed factual description of SkyWeaver
- one selected publication
- clearer project-specific technologies
- clearer date ranges if projects need chronology

### Medium priority missing facts

- whether behavior trees should appear explicitly in the defense version
- whether graph attention is factual for Loyal Wingmen or only a nearby concept
- whether digital twin wording is literally accurate for any current project

### Low priority missing facts

- optional links worth preserving in the final one-page version

## Information That Should Be Removed or De-emphasized

- generic academic wording
- duplicated mentions of “framework” without describing what the framework does
- vague statements that only say “used AI” or “applied machine learning”
- low-priority personal details that do not improve defense-role fit

## Recommended First Resume Narrative

be:

   scenarios.
3. The candidate has direct hands-on work in reinforcement learning, multi-agent
   perception, simulation, and experimentation frameworks.
4. The candidate translates research themes into structured engineering work.
